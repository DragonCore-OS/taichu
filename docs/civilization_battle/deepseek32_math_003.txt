# DeepSeek 3.2 - 数学-3：图论网络流

## 1. 最小费用最大流模型 (MCMF)

### 1.1 线性规划形式

首先将多源多汇问题转化为单源单汇：添加超级源点 S 和超级汇点 T。

**目标函数**：
```
min Σ_(u,v)∈E w(u,v) · f(u,v)
```

**约束条件**：

1. **流量守恒**（除S,T外）：
   ```
   Σ_u f(u,v) - Σ_w f(v,w) = 0, ∀v ∈ V\{S,T}
   ```

2. **源点流出**：
   ```
   Σ_v f(S,v) - Σ_v f(v,S) = F_max
   ```

3. **汇点流入**：
   ```
   Σ_u f(u,T) - Σ_v f(T,v) = -F_max
   ```

4. **容量约束**：
   ```
   0 ≤ f(u,v) ≤ c(u,v)
   ```

其中 F_max 是预先计算的最大流值。

## 2. 最大流最小割定理证明

**定理**：max|f| = min C(S,T)

### 证明：

**上界**：对任意流 f 和割 (S,T)，有 |f| ≤ C(S,T)
- 通过割的净流量 = |f|
- 净流量 ≤ 割的容量（由容量约束）

**下界**：设 f 是最大流，G_f 是残余网络。
- 在 G_f 中不存在从 s 到 t 的增广路
- 定义 S = 在 G_f 中从 s 可达的顶点集，T = V\S
- 对于 (u,v) ∈ S→T：f(u,v) = c(u,v)（满流）
- 对于 (u,v) ∈ T→S：f(u,v) = 0
- 因此 |f| = Σ_{u∈S,v∈T} c(u,v) = C(S,T)

**结论**：max|f| = min C(S,T)。证毕。

## 3. 抗中断备用路径算法

**问题**：已知 k 条边 F ⊆ E 可能中断，预先计算备用路径，使最坏情况下总流量损失最小。

### 近似算法（2-近似）

**步骤**：

1. **计算全网络最大流**：F_total = maxflow(G)
2. **计算可靠网络最大流**：F_safe = maxflow(G \ F)

**决策**：
- 若 F_safe ≥ F_total / 2：
  - 仅使用可靠边传输 F_safe
  - 最坏损失 = 0
  
- 若 F_safe < F_total / 2：
  - 承诺流量 F_commit = F_safe + (F_total - F_safe)/2
  - 路由：F_safe 走可靠边，剩余 ΔF = (F_total - F_safe)/2 走包含F的路径
  - 最坏损失 = ΔF

**近似比证明**：
- 最优损失 L_opt ≥ F_total - F_safe
- 本算法损失 L_algo = (F_total - F_safe)/2 = L_opt / 2
- 因此近似比 ≤ 2

### 备用路径预计算

对于每条脆弱边 e ∈ F，预计算一条不包含 e 的备用路径（使用 Suurballe 算法）：
- 时间复杂度：O(E + V log V) 每条边
- 当 e 中断时，快速切换至备用路径

## 4. 算法复杂度分析

| 算法模块 | 时间复杂度 | 空间复杂度 |
|---------|-----------|-----------|
| MCMF (Successive Shortest Path) | O(F · E log V) | O(V + E) |
| 最大流 (Dinic) | O(V²E) | O(V + E) |
| 最小割构造 | O(VE²) | O(V + E) |
| 抗中断算法 (2-近似) | O(V²E) | O(V + E) |
| 备用路径 (Suurballe) | O(E + V log V) | O(V + E) |

**总体评估**：
- 对于 V=1000, E≈5000-10000 的网络
- Dinic 算法可在秒级完成
- 抗中断策略增加约 100% 开销，仍在可接受范围内

## 5. 文明级洞察

- **冗余即成本**：F_safe 与 F_total 的差距是"安全溢价"，文明需在效率与生存间权衡
- **局部脆弱性**：k条边中断可能引发级联失效，瓶颈往往不在总容量而在关键割集
- **算法伦理**：战争场景下，算法应优先保障基础生存物资，而非单纯追求最大利润
