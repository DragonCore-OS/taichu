## 1. 最小费用最大流模型 (MCMF)

### 1.1 线性规划形式 (Linear Programming Formulation)

**定义**：
- G=(V,E)：供应链网络，|V|=1000
- S⊂V：10 个工厂节点（源点集），供应量 supply(sᵢ)
- T⊂V：50 个零售商节点（汇点集），需求量 demand(tⱼ)
- c_uv：边 (u,v) 的容量
- w_uv：边 (u,v) 的单位运输成本
- f_uv：决策变量，边 (u,v) 上的流量

**目标函数**：最小化总运输成本

Minimize Z = Σ_(u,v)∈E w_uv · f_uv

**约束条件**：

1. **容量约束 (Capacity Constraints)**：
   0 ≤ f_uv ≤ c_uv, ∀(u,v)∈E

2. **流守恒约束 (Flow Conservation)**：
   对于中间节点 v∈V\(S∪T)：
   Σ_(u:(u,v)∈E) f_uv - Σ_(w:(v,w)∈E) f_vw = 0

3. **供需约束 (Supply/Demand Constraints)**：
   - 对于工厂 sᵢ∈S：
     Σ_(w:(sᵢ,w)∈E) f_(sᵢw) - Σ_(u:(u,sᵢ)∈E) f_(usᵢ) = supply(sᵢ)
   - 对于零售商 tⱼ∈T：
     Σ_(u:(u,tⱼ)∈E) f_(utⱼ) - Σ_(w:(tⱼ,w)∈E) f_(tⱼw) = demand(tⱼ)

4. **最大流隐含约束**：
   若总供应 ≠ 总需求，需引入虚拟源点/汇点或允许未满足需求惩罚

## 2. 最大流最小割定理证明

**定理陈述**：在任何网络流图中，从源点 s 到汇点 t 的最大流量等于分离 s 和 t 的最小割容量。

### 2.1 弱对偶性 (Weak Duality)

**引理 1**：对于任意可行流 f 和任意 s-t 割 (S,T)，流值 |f| 不超过割容量 C(S,T)。

**证明**：
根据流守恒，净流出源点集的流量等于总流值：
|f| = Σ_(u∈S,v∈T) f_uv - Σ_(u∈T,v∈S) f_uv

由于 f_uv ≤ c_uv 且 f_uv ≥ 0：
|f| ≤ Σ_(u∈S,v∈T) c_uv = C(S,T)

**结论**：max|f| ≤ min C(S,T)

### 2.2 强对偶性 (Strong Duality)

**引理 2**：存在一个流 f* 和一个割 (S*,T*)，使得 |f*| = C(S*,T*)。

**证明构造**：
1. 运行 Ford-Fulkerson 算法直到无法找到增广路径，得到最大流 f*
2. 构建集合 S*：从源点 s 出发，在残量网络 G_f 中所有可达的节点集合
3. 构建集合 T*：V\S*。由于无法增广，汇点 t∈T*

**分析割边** (u,v) 其中 u∈S*,v∈T*：
- 若 f_uv < c_uv，则残量网络中有边 (u,v)，v 应属于 S*，矛盾。故 f_uv = c_uv（满载）
- 若 f_vu > 0，则残量网络中有边 (v,u)，v 应属于 S*，矛盾。故 f_vu = 0

**计算流值**：
|f*| = Σ_(u∈S*,v∈T*) f_uv - Σ_(u∈T*,v∈S*) f_uv = Σ_(u∈S*,v∈T*) c_uv = C(S*,T*)

**结论**：max|f| = min C(S,T)

## 3. 抗中断备用路径算法

**问题定义**：已知 k 条边集合 K⊂E 可能中断。目标：预先计算流分配，使得最坏情况下总流量损失最小。

### 近似算法：安全核心流加倍策略

**近似比**：≤ 2（相对于最优鲁棒流损失）

### 3.1 算法步骤

1. **计算全网络最大流**：在 G=(V,E) 上计算最大流 F_total
2. **计算安全网络最大流**：在 G'=(V,E\K) 上计算最大流 F_safe（完全不依赖风险边）

**决策阈值**：

- **若 F_safe ≥ (1/2)F_total**：
  - 策略：仅使用安全边 E\K 传输流量 F_safe
  - 最坏情况损失：0（因为不使用风险边）
  - 吞吐量：F_safe

- **若 F_safe < (1/2)F_total**：
  - 策略：使用混合路径
  - 承诺总流量 F_commit = F_safe + (1/2)(F_total - F_safe)
  - 路由：F_safe 部分完全走 E\K，剩余 ΔF = (1/2)(F_total - F_safe) 部分走包含 K 的路径
  - 最坏情况损失：若 K 全部中断，损失仅为 ΔF

### 3.2 近似比证明

**定义损失**：L = F_commit - F_survive

- **最优解下界**：任何策略的存活流量不可能超过 F_safe
- **本算法损失**：
  - 情况 1 (F_safe ≥ F_total/2)：L=0，显然最优
  - 情况 2 (F_safe < F_total/2)：L_algo = 0.5(F_total - F_safe) ≤ (1/2)L_opt_max

### 3.3 备用路径预计算

使用 **Suurballe 算法** 或 **边不相交路径分解**：
- 对于流经 K 的流量部分，为每条风险边 e∈K 预计算一条不包含 e 的备用路径
- 当 e 中断时，快速切换至备用路径

## 4. 算法复杂度分析

| 算法模块 | 算法名称 | 时间复杂度 | 空间复杂度 | 备注 |
|---------|---------|-----------|-----------|------|
| MCMF 求解 | Successive Shortest Path | O(F·ElogV) | O(V+E) | F 为最大流值 |
| 最大流计算 | Dinic 算法 | O(V²E) | O(V+E) | 一般图高效 |
| 最小割构造 | Ford-Fulkerson + BFS | O(VE²) | O(V+E) | 基于最大流残量网络 |
| 抗中断算法 | 2 次 Max Flow + 路径分解 | O(V²E) | O(V+E) | 主要耗时在 2 次最大流计算 |
| 备用路径 | Suurballe 算法 | O(E+VlogV) | O(V+E) | 针对单条边失效的快速重路由 |

### 总体复杂度评估

- 对于 V=1000 的网络，E 通常在 5000～10000 量级
- Dinic 算法运算量约 10⁹～10¹⁰ 次操作，现代 CPU 可在秒级完成
- 抗中断策略仅需额外 1 次最大流计算，开销增加约 100%

## 5. 文明级洞察：供应链韧性

1. **冗余即成本**：F_safe 与 F_total 的差距即为"安全溢价"。文明需在效率与生存之间权衡

2. **局部脆弱性**：k 条边的中断可能引发级联失效。最小割定理告诉我们，瓶颈往往不在总容量，而在关键割集

3. **算法伦理**：在战争或灾难场景下，算法应优先保障"基础生存物资"（高优先级流），而非单纯追求最大利润流
