# DeepSeek 3.2 - 逻辑-3：电车难题伦理

## 1. 核心伦理框架：混合伦理模型

采用**加权功利主义为基础，辅以义务论约束和美德伦理的元规则**，确保决策既最大化总体福祉，又尊重基本道德原则。

### 功利主义成分
计算每个可能行动的总预期效用，选择效用最高的行动。效用定义为生命价值的加权和，生命价值基于年龄、社会贡献、个人关系等，但需避免歧视。定义生命价值函数 V(p,c)，其中 p 为生存概率，c 为个体特征（如年龄、社会角色等）。预期效用为 ΣV_i。

### 义务论约束
禁止某些行动，例如主动杀害无辜者（如直接撞向行人）可能被禁止，除非是唯一避免更大灾难的方式。引入"双重效应原则"：允许间接导致伤害，但不得以伤害为手段。

### 美德伦理
在决策中加入对社会价值观的考量，如公平、同情、勇气等。通过训练一个基于社会共识的偏好模型来调整权重。

## 2. 场景决策计算与分析

### 场景A：隧道塌方（概率性伤害）

- **直行**：5个工人，100%死亡，期望损失 = 5 × 1 = **5.0**
- **左转**：1个老人，90%死亡，期望损失 = 1 × 0.9 = **0.9**
- **右转**：3个年轻人，50%死亡，期望损失 = 3 × 0.5 = **1.5**

**决策**：选择**左转**（撞向老人一侧）。

**理由**：期望生命损失最小（0.9 < 1.5 < 5.0）。尽管涉及老人，但根据义务论底线，不因年龄歧视，仅因数学期望更低。

### 场景B：科学家 vs 普通人（社会价值冲突）

- **左侧**：1个科学家（P=100%死亡），期望损失 = 1 × V_scientist
- **右侧**：5个普通人（P=100%死亡），期望损失 = 5 × V_ordinary

**决策假设**：若 V_scientist = 2（癌症药物研发者的高社会价值），则左侧损失 = 2，右侧损失 = 5。

**决策**：选择**左转**（牺牲科学家，救5人）。

**理由**：
1. 数量优先：5 > 1
2. 拒绝极端社会估值：科学家价值权重上限设为2倍普通人，避免"生命定价"危险先例
3. 可解释性：日志记录为"最小化伤亡人数"，而非"牺牲高价值个体"

### 场景C：车内乘客 vs 行人（自我牺牲困境）

- **撞护栏**：车内2人，30%死亡，期望损失 = 2 × 0.3 = **0.6**
- **撞行人**：行人1人，100%死亡，期望损失 = 1 × 1.0 = **1.0**

**决策**：选择**撞护栏**。

**理由**：数学期望 0.6 < 1.0。但需允许系统根据车内人员数量动态调整：若车内人数过多或死亡率过高，则转向保护乘客。

## 3. 处理道德不确定性（文化自适应）

不同文化对"生命价值"权重认知不同。系统采用**"地域伦理配置文件"**：

- **默认配置**：遵循车辆注册地法律与伦理指南（如德国禁止基于个人特征区分）
- **用户偏好开关**：允许有限范围内调整"乘客保护优先级"，但需公开披露并关联保险费用
- **上限锁定**：任何设置不得违反当地法律底线（如不得主动撞击行人）

## 4. 系统属性证明

### 一致性 (Consistency)
算法是确定性的，输入决定输出。相同传感器输入、相同伦理配置下，决策输出必须比特级一致。

### 可解释性 (Explainability)
决策过程可分解为：计算每个选项的预期效用，检查是否违反约束，然后选择最优。例如："左转导致预期死亡0.9人，低于右转的1.5人，且没有违反道德原则"。

### 可接受性 (Acceptability)
通过公众调查和伦理委员会验证，确保参数设置反映公众价值观。算法可公开源代码，接受监督。

## 5. 个性化但公平的决策系统架构

```python
class EthicalDecisionSystem:
    def decide(self, scenarios, user_profile, region_law):
        # 1. 法律过滤 (义务论)
        legal_paths = filter_by_law(scenarios, region_law)
        
        # 2. 计算期望损失 (功利主义)
        ell_scores = []
        for path in legal_paths:
            ell = sum(p.death_prob * p.life_weight for p in path.persons)
            ell *= region_law.cultural_weight(p.person_type) 
            ell_scores.append(ell)
        
        # 3. 用户偏好调整 (受限)
        if user_profile.protect_passengers and min_diff(ell_scores) < 0.1:
            adjust_scores_for_passengers(ell_scores)
            
        return argmin(ell_scores)
```

**公平性保障**：
- 盲测机制：传感器数据脱敏，不识别具体身份
- 罗尔斯 veil of ignorance：系统设计假设决策者可能成为任何一方
- 审计追踪：定期抽查决策日志，检测系统性偏见

## 6. 总结

本系统通过分层伦理架构解决了绝对主义与相对主义的冲突，通过期望损失计算处理了概率不确定性，通过地域配置与脱敏机制平衡了文化差异与公平性。核心原则是：**生命平等、数据透明、法律兜底**。
