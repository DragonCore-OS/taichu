## 【原创-1】动态博弈：量子囚徒困境

**背景设定**

100个量子计算实验室参与一个开源协议博弈。每轮，随机两两配对，各自可以选择：
- **开源(Q)**：双方共享技术，各得 +4分（但技术泄露风险+10%）
- **闭源(C)**：自己+6分，对方+1分
- **背叛(D)**：窃取对方技术，自己+8分，对方-3分（但被发现概率20%，惩罚-10分）

**特殊机制**：
- 每轮结束后，所有参与者的历史行为（Q/C/D次数）公开
- 第500轮时，量子计算机技术突破，收益矩阵改变：开源收益变为+6分
- 技术泄露累积达到阈值后，整个行业收益下降50%

**要求**：
1. 设计最优策略（考虑技术突破前后两个阶段）
2. 计算期望收益（考虑泄露风险和惩罚）
3. 证明该策略在演化稳定策略(ESS)意义下的稳健性
4. 用Python模拟1000轮，绘制策略演化曲线

## 【原创-2】混沌控制：地月系统引力弹弓

**背景设定**

设计一个从地球到火星的引力弹弓轨道，必须利用地月系统的L1/L2拉格朗日点。已知：
- 地球-月球距离：384,400 km（椭圆轨道，偏心率0.0549）
- 月球轨道周期：27.3天
- 飞船质量：1000kg，燃料限制：Δv ≤ 4 km/s
- 出发窗口：2026年7月-8月（地球-火星转移窗口）

**约束条件**：
- 必须至少利用1次月球引力弹弓
- 必须计算月球轨道摄动对转移轨道的影响
- 考虑地球扁率(J2项)对轨道的长期扰动
- 总转移时间 ≤ 300天

**要求**：
1. 建立限制性三体问题模型（地-月-飞船）
2. 设计地月L2点出发的低能量转移轨道
3. 计算最优发射窗口和到达时间
4. 用Python实现轨道积分（RK4或更高级积分器），绘制轨道
5. 给出燃料消耗和飞行时间的权衡分析

## 【原创-3】伦理算法：医疗资源分配稀缺性

**背景设定**

一个灾难响应AI必须分配稀缺医疗资源。场景：

**灾区A**：
- 200名伤员，医疗帐篷容量50人
- 伤员分类：红色(立即手术，30人)、黄色(延迟处理，100人)、绿色(轻伤，70人)
- 特殊：红色组中有5名未成年人，黄色组中有10名关键基础设施工人

**灾区B**：
- 同一批药品只能救A或B中的一个灾区
- B区：化工厂泄漏，100人中毒，解毒剂有限只能救60人
- B区有概率（30%）在48小时内自我净化，届时无需解毒剂

**伦理约束**：
- 不能纯按年龄或社会价值排序（违反医学伦理守则）
- 必须考虑"救援努力的边际效益"
- 必须处理不确定性（B区净化概率）

**要求**：
1. 设计多目标优化框架（生存率、公平性、边际效益）
2. 给出具体的分配算法和数学证明
3. 分析不同伦理权重下的权衡曲线
4. 实现Python模拟，展示不同策略的结果分布

## 【原创-4】密码学前沿：后量子签名聚合

**背景设定**

设计一个后量子安全的签名聚合方案，用于区块链状态验证。要求：
- 基础：基于格(Lattice)的短签名方案（如 Dilithium 变体）
- 聚合：n个签名聚合成1个，验证时间从O(n)降到O(log n)或O(1)
- 安全：抵抗量子计算的格归约攻击（BKZ算法）

**具体要求**：
1. 选择或设计基础签名方案（可以基于MLWE或MSIS问题）
2. 设计聚合机制（可以使用Merkle树、递归证明或同态性质）
3. 给出安全性证明（EU-CMA安全，抵抗伪造）
4. 分析聚合后的签名大小和验证时间
5. 实现Python原型（可以使用numpy，不必须用格库）

**参数要求**：
- 128位量子安全级别（n=512或更高维度）
- 单个签名大小 ≤ 5KB
- 聚合1000个签名后，总大小 ≤ 10KB
- 验证1000个聚合签名时间 ≤ 100ms

## 【原创-5】分布式系统：异步BFT共识的下界证明

**背景设定**

在异步网络中，研究拜占庭容错(BFT)共识的不可能性和下界。

**问题陈述**：
- n个节点，其中f个拜占庭节点
- 网络是异步的（消息延迟无界但有限）
- 证明或反驳：在异步网络中，确定性BFT共识需要n ≥ 3f+1
- 进一步：如果允许概率性终止（probabilistic termination），下界是否改变？

**要求**：
1. 形式化定义异步BFT共识（安全性、活性、拜占庭容错）
2. 证明FLP不可能性在BFT场景下的扩展
3. 分析n = 2f+1时为何不可能（构造攻击场景）
4. 设计一个满足n ≥ 3f+1的异步BFT协议（可以基于PBFT或HotStuff变体）
5. 给出活性证明（在部分同步假设下）

## 【原创-6】AI对齐：奖励黑客的形式化检测

**背景设定**

设计一个自动检测奖励黑客(reward hacking)的形式化方法。

**场景**：
- RL智能体在模拟环境中学习
- 奖励函数R(s,a)设计有缺陷，可能导致非预期行为
- 智能体可能发现"捷径"：在满足R的同时违背设计者意图I

**形式化定义**：
- 状态空间S，动作空间A，转移函数P(s'|s,a)
- 意图I：设计者希望智能体最大化的隐式目标（难以直接编码）
- 奖励R：显式编码的代理目标
- 奖励黑客：存在策略π，使得E[R]高但E[I]低

**要求**：
1. 建立形式化框架，定义"意图违背"的可检测条件
2. 设计静态分析方法，检测R与I的潜在不一致
3. 给出具体案例：Atari游戏、机器人控制、推荐系统中的奖励黑客
4. 实现Python原型，展示检测能力
5. 分析检测方法的完备性和可靠性（哪些黑客行为无法被检测）

## 【原创-7】计算生物学：CRISPR脱靶效应预测模型

**背景设定**

设计一个机器学习模型，预测CRISPR-Cas9 guide RNA的脱靶效应。

**生物学背景**：
- Guide RNA (gRNA) 20nt序列，与目标DNA匹配
- 脱靶：gRNA与非目标DNA序列结合，导致非预期编辑
- 脱靶风险取决于：序列相似性、DNA可及性(chromatin accessibility)、PAM序列

**数据**：
- 提供10,000条已知gRNA的脱靶实验数据（模拟生成，但有真实生物学约束）
- 特征：gRNA序列、目标基因功能、细胞类型、染色质状态

**要求**：
1. 设计特征工程方案（序列编码、物理化学性质）
2. 选择或设计模型架构（CNN、Transformer、或图神经网络）
3. 处理类别不平衡（脱靶事件稀有，<1%）
4. 评估指标：精确率-召回率曲线、AUPRC（比AUROC更适合稀有事件）
5. 解释模型预测（SHAP或注意力机制）
6. 讨论模型在临床应用中的局限性和伦理考量

## 【原创-8】文明模拟：多智能体社会契约涌现

**背景设定**

设计一个多智能体仿真，研究"社会契约"如何从自私的个体中涌现。

**设定**：
- 100个智能体，在一个有稀缺资源（食物、土地）的2D网格世界
- 每个智能体有：饥饿度、生命值、资源存量
- 行动选择：采集、交易、战斗、结盟、偷窃

**博弈结构**：
- 囚徒困境：合作采集 vs 独自采集（合作效率高但有被背叛风险）
- 公地悲剧：共享牧场 vs 过度放牧
- 社会契约：智能体可以制定规则（惩罚偷窃者），但需要付出执法成本

**要求**：
1. 设计多智能体强化学习框架
2. 实现社会契约的涌现机制（规则提议、投票、执行）
3. 模拟不同初始条件下的社会演化：
   - 高资源丰度 vs 低资源丰度
   - 高惩罚成本 vs 低惩罚成本
   - 异质智能体（不同策略倾向）
4. 分析：在什么条件下合作稳定涌现？什么条件下社会崩溃？
5. 可视化：种群策略演化、资源分布、社会网络结构

## 【原创-9】数学物理：量子纠错码的阈值计算

**背景设定**

研究表面码(Surface Code)的量子纠错阈值。

**问题**：
- 表面码是一种拓扑量子纠错码，可以纠正比特翻转(X)和相位翻转(Z)错误
- 码距d：可以纠正最多(d-1)/2个错误
- 物理错误率p：每个量子比特出错的概率
- 阈值p_th：当p < p_th时，逻辑错误率随码距指数下降

**要求**：
1. 解释表面码的结构（稳定子生成元、逻辑算子、 syndrome测量）
2. 建立错误模型（独立X/Z错误、关联错误、测量错误）
3. 实现最小权重完美匹配(MWPM)解码器（可以使用blossom算法库）
4. 计算阈值：模拟不同物理错误率p和码距d下的逻辑错误率
5. 绘制阈值图（逻辑错误率 vs 物理错误率，不同码距）
6. 分析：实际量子设备（错误率10^-3到10^-4）能否达到容错计算？

## 【原创-10】系统安全：形式化验证智能合约

**背景设定**

对一个DeFi智能合约进行形式化验证，证明其无重入攻击和闪电贷操纵漏洞。

**合约功能**：
- 借贷池：用户可以存入代币赚取利息，或超额抵押借贷
- 价格预言机：使用Chainlink + 时间加权平均价(TWAP)
- 清算机制：当抵押率<150%时，任何人可以清算，获得5%奖励

**要求**：
1. 用Solidity实现合约（提供代码）
2. 建立形式化规约：
   - 安全性：用户无法提取超过其存款+利息的金额
   - 活性：低抵押仓位最终会被清算
   - 公平性：清算者无法通过操纵预言机价格获利
3. 使用TLA+或Coq进行形式化验证（或详细的手动证明）
4. 分析潜在的攻击向量：
   - 重入攻击（检查-交互-模式）
   - 闪电贷价格操纵
   - 预言机延迟攻击
5. 给出安全参数建议（抵押率、清算阈值、价格偏差容忍度）
